Common Responsive Patterns

Intro To Patterns

In this portion of the lesson I was Introduced to Patterns.I learned that established patterns that work well, across any device can be categorized into one of four patterns.Mostry fluid,column drop, layout shifter, and off canvas, and in some cases a wepage will use combination of two.

Column Drop

In this portion of the Course I was Introduced to the Column drop pattern.I learned that out of the four patterns Column Drop is probably the easiest.At it's narrowest viewport, each element simply stacks vertically,one on top of the other, As things get wider the elements expand, until the first break point is hit. At the firts breakpoint, instead of having all the elements stacked. Now there are two elements stacked above one element. Once that breakpoint is hit all three elements are together side by side.

Mostly Fluid

The mostly fluid pattern is very similar to the Column drop pattern with more columns and columns fitting in different ways, depending on the viewport width.just like column drop at its narrowest viewport, the layout is stacked but as the layout gets wider, the grid pattern starts to appear. and eventually once the layout hits it's widest viewport, margins are added on the left and right, instead of expanding things out.The instructor of the course then went over how to utilize this pattern by using it in css.

Layout Shifter

The layout shifter pattern is probably the most responsive pattern with multiple breakpoints across several differen screen widths,but the key to this layout is the way that contents moves about instead of reflowing and dropping below other columns.Flex box really shines here, because we can use the order CSS attribute.Because so much can change in each layout, it does require a bit more planning to maintain.

Off Canvas

In this portion of the lesson the instructor went into depth with the off canvas pattern.With off canvas, instead of stacking content vertically, the off canvas places less frequently used content, for example navigation or app menus, off screen, only showing them if the screen is large enough.On smaller screens, the off canvas content is typically shown when the user taps on the icon.I was then given some quizes to test out my new skills.

Optimizations

Images

Images play an integral part of every website.The HTTP archive has it pegged at near 1.2 megabytes per page,Which means that images take up 65% of every website.In order to do responsive web design, we need to consider images.If the rest of my page is changing based on device characteristics.Using the same image but changing the resolution is probably one of the most common cases.Think of providing a standard image for a labtop, but a 2X image for a high DPI display,like a Chromebook Pixel, or a Retina iMac.The best way to do this is by using the source set attribute on an image tag.With source set, the browser will choose which file it wants, and only download that one.

Responsive Tables Intro

Tables can be tricky if they have more than a few columns, theres a pretty good chance that it's going to overflow the viewport on smaller screens, forcing horizontal scrolling. In this course the instructor went over three methods that are commonly used. Hidden Columns, No more tables and Contained tables. Like with breakpoints you want to experiment to decide what works best for your web sites content.

Hidden Columns

Hidden columns essentially hides columns based on importance as viewport size get's smaller.I was then given a quiz to apply what i had just learned about hidden columns.

No More Tables

With the no mor tables technique, below a certain viewport width,the table is collapsed and resembles a long list, as opposed to a table data.The nice thing about this technique is all of the data is visible no matter what the size of the viewport is.
